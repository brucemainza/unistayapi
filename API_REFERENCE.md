# UniStay API Reference

Generated by exercising the running FastAPI application against an in-memory SQLite test database with seed data. Every request/response example below was captured from a real call.

**Approach used:** the app was run via `httpx.ASGITransport` using the same fixtures as the pytest suite (`scripts/capture_api_responses.py`). External service behavior reflects the credentials currently configured in `.env`.

---

## 1. Overview & Conventions

### Base URLs

| Environment | URL |
|---|---|
| Local development | `http://localhost` (nginx reverse proxy) or `http://localhost:8000` (direct) |
| Render production | `https://<your-service>.onrender.com` |

All endpoints documented here are mounted under `/api`.

### Global Response Envelope

Almost every endpoint wraps its payload in:

```json
{
  "status": true,
  "message": "...",
  "data": { ... } | [ ... ] | null
}
```

Errors use the same envelope with `status: false`:

```json
{
  "status": false,
  "message": "Authentication failed",
  "data": null
}
```

### Authentication

- Obtain a token from `POST /api/auth/register` or `POST /api/auth/login`.
- Send the token on protected endpoints as: `Authorization: Bearer <token>`.
- Token expiry is controlled by `JWT_EXPIRES_IN` (default 86400 seconds = 24 hours).
- There is **no refresh-token flow** currently implemented.
- In non-production environments only, the literal token `dev-student-token` is accepted and resolves to a test student user.

### Field Naming Convention — Important for Flutter

The API is **inconsistent** across domains:

| Domain | Request keys | Response keys |
|---|---|---|
| Auth / Users | `snake_case` | `snake_case` |
| Houses / Rooms / Favorites / Universities / Landlords / Payments / Notifications | `snake_case` | `camelCase` |
| Bookings | `snake_case` | `camelCase` |

Examples of response keys you will actually receive:

- Houses: `universityId`, `walkTime`, `driveDistance`, `availableSpaces`, `imageUrls`, `paymentMethods`, `nearbyUniversities`
- Rooms: `houseId`
- Bookings: `studentId`, `houseId`, `roomId`, `moveInDate`, `houseName`, `roomType`, `createdAt`, `updatedAt`
- Notifications: `isRead`, `createdAt`

**Flutter/Dart implication:** you need a mapping/serialization layer; do not assume all endpoints return snake_case.

### Date/Time Format

- Dates come back as ISO 8601 strings.
- **Inconsistency:** some timestamps include the timezone offset (`2026-07-11T09:03:27.853387+00:00`) and some omit it (`2026-07-11T09:03:27.853387`).
- Booking `moveInDate` is returned as a plain date string: `2026-08-01`.

### Standard Headers

- `Content-Type: application/json` for JSON bodies.
- `Authorization: Bearer <token>` for protected endpoints.
- The API returns `x-request-id` in every response header for tracing.

#### `POST /api/auth/verify-otp`

Verifies the 5-digit phone OTP. In non-production environments any 5-digit code is accepted.

- **Auth:** none
- **Request**

```json
{
  "user_id": "fcc88a43-d1e7-49ef-8647-467d110bcac1",
  "code": "12345"
}
```

- **Success 200** — returns token and user (same shape as register/login).

- **Error 422** — code not 5 digits

```json
{
  "status": false,
  "message": "OTP must be a 5-digit code",
  "data": null
}
```

#### `POST /api/auth/resend-otp`

- **Auth:** none
- **Request**

```json
{
  "user_id": "fcc88a43-d1e7-49ef-8647-467d110bcac1"
}
```

- **Success 200**

```json
{
  "status": true,
  "message": "OTP resent",
  "data": {
    "code": "12345",
    "expires_in": 600
  }
}
```

In production this only creates a new OTP record; the actual SMS delivery is not wired up.

#### `POST /api/auth/signup`

Email/OTP signup path. Creates an unverified user and triggers an email OTP in a background task.

- **Auth:** none
- **Request**

```json
{
  "full_name": "Signup User 2674538291",
  "phone": "2674538291",
  "email": "signup_2674538291@example.com",
  "password": "secret123",
  "role": "student"
}
```

- **Success 201**

```json
{
  "status": true,
  "message": "Verification code sent",
  "data": {
    "id": "...",
    "email": "signup_2674538291@example.com"
  }
}
```

- **Rate limit:** 5 attempts per IP per 60 seconds. **Error 429:**

```json
{
  "status": false,
  "message": "Too many signup attempts; please try again later",
  "data": null
}
```

If `RESEND_API_KEY` is not configured, the email is silently skipped and a warning is logged.

#### `POST /api/auth/resend-email-otp`

- **Auth:** none
- **Request**

```json
{
  "email": "signup_2674538291@example.com"
}
```

- **Success 200**

```json
{
  "status": true,
  "message": "Verification code sent",
  "data": null
}
```

- **Error 429** — resend cooldown active immediately after signup

```json
{
  "status": false,
  "message": "Please wait before requesting another code",
  "data": null
}
```

#### `POST /api/auth/verify-email`

- **Auth:** none
- **Request**

```json
{
  "email": "signup_2674538291@example.com",
  "code": "000000"
}
```

- **Error 401** — wrong/expired code (captured example)

```json
{
  "status": false,
  "message": "Invalid or expired code",
  "data": null
}
```

- **Success 200** — returns token and user (same shape as register).

## 4. Flutter Integration Notes

### Authentication flow

1. Call `POST /api/auth/signup` to create an account and receive an email OTP.
2. Call `POST /api/auth/verify-email` with the code from email.
3. Use the returned token on all protected endpoints.
4. Alternatively, use `POST /api/auth/register` + `POST /api/auth/verify-otp` for the phone-OTP flow.
5. There is no refresh token — store the JWT and re-login when it expires.

### Mapping layer

You will need two serialization strategies:

- **Snake_case responses:** Auth and Users endpoints.
- **CamelCase responses:** Houses, Rooms, Bookings, Favorites, Payments, Notifications, Landlords.

Request bodies are consistently `snake_case` across all endpoints.

### Nullable fields

The following fields can be `null` and must be modeled as optional:

- House: `formattedAddress`, `universityId` (for houses not tied to a university)
- Booking: `note`, `houseName`, `roomType`
- User: `email_verified` (false until verified)

### External-dependency endpoints

Do not build core flows against these until the backend credentials are fixed:

- `GET /api/houses/{id}/eta`
- `GET /api/places/autocomplete`
- `GET /api/places/details`
- `POST /api/images/upload`
- `DELETE /api/landlords/houses/{id}` (when the house has rooms)

### Testing on Render

Use the same curl examples from the README, but against `https://your-service.onrender.com`. Remember that production requires `GOOGLE_MAPS_SERVER_KEY`, `REDIS_URL`, and Cloudinary credentials for the endpoints above to work.

## 5. Not Yet Implemented / Do Not Integrate Against

These routes are wired into the router but failed when exercised end-to-end with the current configuration/code.

### `GET /api/houses/{id}/eta`

- **What it should do:** return cached or fresh drive/walk ETA from a university to a house.
- **Why it failed:** the configured Google Maps key does not have access to the Routes API; returns HTTP 502.
- **Real error captured:**

```json
{
  "status": false,
  "message": "Routes API failed (HTTP 403)",
  "data": null
}
```

### `GET /api/places/autocomplete`

- **What it should do:** proxy Google Places autocomplete.
- **Why it failed:** the configured Google Maps key does not have access to the Places API (New); returns HTTP 502.

### `GET /api/places/details`

- **What it should do:** proxy Google Place details.
- **Why it failed:** same Places API key restriction.

### `POST /api/images/upload`

- **What it should do:** upload an image to Cloudinary and return its URL.
- **Why it failed:** the configured Cloudinary credentials fail (`Cloudinary upload failed: Failed to ping image`); returns HTTP 502.
- **Note:** file-type validation works and returns 422 for non-image uploads.

### `DELETE /api/landlords/houses/{id}`

- **What it should do:** delete a house and its rooms.
- **Why it failed:** the cascade delete is broken. The service tries to set `rooms.house_id` to `NULL`, violating the `NOT NULL` constraint, and returns HTTP 500.
- **Real error captured:**

```json
{
  "exception": "IntegrityError",
  "detail": "(sqlite3.IntegrityError) NOT NULL constraint failed: rooms.house_id ..."
}
```
